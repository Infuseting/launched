Design: Data Flow & Error Handling

Ensuring robustness and efficiency in the launcher's lifecycle.

Data Flow
  1. Bootstrap: Check for launcher updates. If available, self-update and restart.
  2. Session Discovery: Fetch servers.json. Determine current session (last used or isDefault: true).
  3. UI Setup: Load htmlPath into WebView. Inject Bridge Script.
  4. Authentication: Execute AuthStrategy (Microsoft/SSO/etc.).
  5. Sync: Fetch syncUrl manifest. Compare MD5s in syncDir. Download/Cleanup as needed.
  6. Installation: Check .minecraft/versions for required Minecraft/Forge JARs. Download if missing.
  7. Launch: Build JVM arguments (merging global and session-specific overrides). Spawn Java process.

Error Handling Scenarios

  • Invalid Session: Both Forge and Fabric specified? Log error and fallback to next session/default.
  • Network Failure: Remote htmlPath unreachable? Fallback to local default index.html.
  • Sync Error: MD5 mismatch or download failure? Alert user and prevent game launch.
  • Launch Failure: Invalid JVM arguments or missing Java? Display logs if "with logs" enabled.

Sync Logic Detail

Defining how the syncDir and syncUrl interact to keep files updated.

Sync Process Flow
  1. Fetch servers.json
  2. For current session, fetch syncUrl metadata
  3. Identify local files in syncDir folders (e.g., mods/)
  4. Match & Update: Compare local MD5 with server MD5. Download if different or missing.
  5. Cleanup: If a local file is in a syncDir folder but NOT in the server manifest (and not excluded via !), delete it.

Is this cleanup logic correct?

Example: syncDir = "mods". A user manually adds Optifine.jar to their local mods folder. If Optifine.jar is NOT in the syncUrl manifest, the launcher will automatically delete it on start.
